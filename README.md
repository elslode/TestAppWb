# _test_app_wb

## Архитектурный подход

В проекте используется **BLoC (Business Logic Component)** как архитектурный паттерн и библиотека `flutter_bloc`.

### Основные принципы:
- Единый источник истины — всё состояние хранится в одном BLoC
- UI-экраны не содержат бизнес-логики
- Навигация управляется состоянием
- Состояние переживает переходы между шагами

---

## Управление состоянием

### FormFlowBloc
Отвечает за:
- текущий шаг (`currentStep`)
- все данные формы
- переходы вперёд и назад
- сброс процесса

### FormFlowState
Хранит:
- текущий шаг
- значения всех полей формы
- используется как единое состояние всего flow

### FormFlowEvent
Описывает намерения пользователя:
- изменение данных
- переходы между шагами
- сброс процесса
