# testappwb

Flutter-приложение с пошаговым вводом данных (wizard / stepper), реализованное в рамках тестового задания.

Цель проекта — продемонстрировать:
- понимание архитектуры Flutter-приложений
- работу с состоянием через BLoC (flutter_bloc)
- корректную навигацию вперёд / назад
- аккуратное разделение UI и бизнес-логики

Дизайн не является приоритетом — фокус сделан на структуре и логике.

---

## Общая идея приложения

Приложение представляет собой пошаговый процесс ввода данных, состоящий из **4 экранов**:

1. Ввод базовой информации
2. Выбор параметров
3. Дополнительные данные
4. Итоговый экран с результатами

Навигация между шагами реализована без использования готовых степперов.

Все введённые данные:
- сохраняются в состоянии
- не теряются при переходе назад
- могут быть полностью сброшены и начаты заново

---

## Архитектурный подход

В проекте используется **BLoC (Business Logic Component)** как архитектурный паттерн и библиотека `flutter_bloc`.

### Основные принципы:
- **Единый источник истины** — всё состояние хранится в одном BLoC
- UI-экраны не содержат бизнес-логики
- Навигация управляется состоянием, а не Navigator.push/pop
- Состояние переживает переходы между шагами

---

## Управление состоянием

### FormFlowBloc
Отвечает за:
- текущий шаг (`currentStep`)
- все данные формы
- переходы вперёд / назад
- сброс процесса

### FormFlowState
Хранит:
- текущий шаг
- значения всех полей формы
- используется как единое состояние всего flow

### FormFlowEvent
Описывает намерения пользователя:
- изменение данных
- переходы между шагами
- сброс процесса

---

## Навигация между экранами

Навигация реализована через `IndexedStack`, управляемый значением `currentStep` из состояния BLoC.

Почему выбран этот подход:
- экраны не уничтожаются при переходах
- данные не теряются при возврате назад
- навигация полностью контролируется бизнес-логикой

---

## Описание экранов

### Step 1 — Базовые данные
- TextField: имя
- TextField: email
- Кнопка «Далее»

Данные сразу отправляются в BLoC через события.

---

### Step 2 — Параметры
- DropdownButtonFormField: тип пользователя
- Checkbox: согласие с условиями
- Кнопки «Назад» / «Далее»

---

### Step 3 — Дополнительно
- TextField: комментарий
- Кнопки «Назад» / «Далее»

---

### Summary — Итоговый экран
- Отображение всех введённых данных в формате «ключ – значение»
- Кнопка «Повторить процесс»
- При повторе состояние полностью сбрасывается и пользователь возвращается на первый шаг

---

## Сброс состояния

Сброс процесса реализован через событие `ResetFormPressed`, которое:
- возвращает состояние к начальному
- очищает все введённые данные
- возвращает пользователя на первый шаг

Для корректного отображения сброса в UI используются `TextEditingController` и реакция на изменения состояния.
